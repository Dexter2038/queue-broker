# queue-broker

Простой брокер очередей в памяти на Go с HTTP API.

## Возможности

- Очереди FIFO в памяти
- Потокобезопасность, поддержка одновременных запросов
- Поддержка таймаута для блокирующего GET
- Настраиваемый порт
- Только стандартная библиотека Go

## API

### PUT `/{queue}?v=message`

Добавляет сообщение в очередь с именем `queue. Возвращает 200 OK при успехе, 400 Bad Request, если параметр `v` отсутствует.

Пример:
    curl -XPUT http://localhost:8080/pet?v=cat

### GET `/{queue}`

Забирает сообщение из очереди в порядке FIFO. Возвращает тело сообщения и 200 OK, либо 404 Not Found, если очередь пуста.

Необязательный параметр `timeout`: `GET /{queue}?timeout=N`. Если очередь пуста, запрос ожидает до `N` секунд появления сообщения. При истечении таймаута возвращается 404.

Пример:
    curl http://localhost:8080/pet
    curl http://localhost:8080/pet?timeout=5

## Сборка и запуск

    go build -o queue-broker
    ./queue-broker -port=8080

По умолчанию используется порт 8080.

## Тестирование

Используйте curl-команды, как показано. Несколько клиентов могут одновременно ожидать сообщения; первый в очереди ожидающих получит первое доступное сообщение.

## Контекст задания

Проект реализован в рамках тестового задания на позицию Go-разработчика. Соответствует всем требованиям: только стандартная библиотека, один файл, лаконичный код.
